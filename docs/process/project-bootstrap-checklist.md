# Project Bootstrap Checklist

Use this checklist when creating a new repository from this foundation.

## Ownership and governance

- [ ] Replace placeholder entries in `CODEOWNERS`
- [ ] Configure branch protection for `main`
- [ ] Require PR reviews and status checks

## CI and quality gates

- [ ] Keep foundation hygiene checks enabled
- [ ] Add language/framework-specific lint steps
- [ ] Add project-specific test/build steps
- [ ] Mark required checks in branch protection

## Documentation setup

- [ ] Update `README.md` with project purpose and setup
- [ ] Update `ARCHITECTURE.md` with project-specific architecture notes
- [ ] Keep standards/process docs aligned with team practices

## Security and support

- [ ] Define security contact and reporting path in `SECURITY.md`
- [ ] Define support channels and ownership in `SUPPORT.md`

## Delivery readiness

- [ ] Verify PR and issue templates match team workflow
- [ ] Ensure testing strategy covers initial product surface
- [ ] Confirm human review policy before first production merge
