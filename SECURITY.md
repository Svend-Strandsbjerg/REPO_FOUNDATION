# Security Policy

## Supported versions

This repository is a foundation template. Downstream projects should define and maintain explicit supported-version policies based on their release strategy.

## Reporting a vulnerability

If you discover a security issue in a repository built from this foundation:

1. Do **not** open a public issue with exploit details.
2. Report privately to the maintainers/security contact configured by the project owners.
3. Include reproduction details, impact, and suggested remediation where possible.

## Maintainer expectations

Project maintainers should:

- Define a private security contact channel.
- Triage reports promptly.
- Communicate remediation timelines clearly.
- Publish advisories when appropriate.
